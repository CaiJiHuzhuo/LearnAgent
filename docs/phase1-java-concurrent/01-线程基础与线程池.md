## 一、线程基础

### 1.1 进程与线程

| 概念 | 说明 |
|------|------|
| 进程（Process） | 操作系统资源分配的最小单位，拥有独立内存空间 |
| 线程（Thread） | CPU 调度的最小单位，共享进程内存空间 |
| 协程（Coroutine） | 用户态轻量级线程，由程序自行调度（Java 21 虚拟线程） |

### 1.2 线程上下文切换

线程切换需要保存/恢复寄存器、程序计数器、栈信息等，代价较高。  
**减少上下文切换的策略：**
- 使用无锁并发（CAS）
- 减少线程数量，使用线程池
- 使用协程/虚拟线程

## 二、线程生命周期

```
NEW → RUNNABLE → (BLOCKED / WAITING / TIMED_WAITING) → TERMINATED
```

| 状态 | 说明 |
|------|------|
| `NEW` | 线程已创建，尚未调用 `start()` |
| `RUNNABLE` | 可运行状态（含就绪和运行中） |
| `BLOCKED` | 等待 synchronized 锁 |
| `WAITING` | 调用 `wait()` / `join()` / `park()` 无限等待 |
| `TIMED_WAITING` | 带超时的等待，如 `sleep(ms)` |
| `TERMINATED` | 线程执行完毕 |

```java
public class ThreadStateDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread t = new Thread(() -> {
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });

        System.out.println(t.getState()); // NEW
        t.start();
        System.out.println(t.getState()); // RUNNABLE or TIMED_WAITING
        t.join();
        System.out.println(t.getState()); // TERMINATED
    }
}
```

## 三、创建线程的方式

### 3.1 继承 Thread 类

```java
public class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread: " + Thread.currentThread().getName());
    }

    public static void main(String[] args) {
        new MyThread().start();
    }
}
```

### 3.2 实现 Runnable 接口（推荐）

```java
public class RunnableDemo implements Runnable {
    @Override
    public void run() {
        System.out.println("Runnable: " + Thread.currentThread().getName());
    }

    public static void main(String[] args) {
        Thread t = new Thread(new RunnableDemo());
        t.start();

        // Lambda 简写
        new Thread(() -> System.out.println("Lambda thread")).start();
    }
}
```

### 3.3 实现 Callable + FutureTask（有返回值）

```java
import java.util.concurrent.*;

public class CallableDemo implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {
        Thread.sleep(500);
        return 42;
    }

    public static void main(String[] args) throws Exception {
        FutureTask<Integer> task = new FutureTask<>(new CallableDemo());
        new Thread(task).start();
        System.out.println("结果: " + task.get()); // 阻塞等待结果
    }
}
```

### 3.4 线程池提交任务（生产推荐）

```java
ExecutorService pool = Executors.newFixedThreadPool(4);
Future<String> future = pool.submit(() -> "hello from pool");
System.out.println(future.get());
pool.shutdown();
```

## 四、线程池核心原理

### 4.1 为什么要用线程池？

1. **降低资源消耗**：复用线程，减少创建/销毁开销
2. **提高响应速度**：任务到来时无需等待线程创建
3. **提高线程可管理性**：统一调度、监控、限流

### 4.2 线程池工作流程

```
提交任务
    │
    ▼
核心线程数是否已满？
    │ 否                  是
    ▼                     ▼
创建核心线程执行    队列是否已满？
                     │ 否        是
                     ▼           ▼
                  放入队列  线程总数是否已达 maximumPoolSize？
                               │ 否             是
                               ▼                ▼
                          创建非核心线程    执行拒绝策略
```

## 五、ThreadPoolExecutor 详解

### 5.1 构造参数

```java
public ThreadPoolExecutor(
    int corePoolSize,          // 核心线程数
    int maximumPoolSize,       // 最大线程数
    long keepAliveTime,        // 非核心线程空闲存活时间
    TimeUnit unit,             // 时间单位
    BlockingQueue<Runnable> workQueue,  // 工作队列
    ThreadFactory threadFactory,       // 线程工厂
    RejectedExecutionHandler handler   // 拒绝策略
)
```

### 5.2 工作队列类型

| 队列类型 | 特点 |
|----------|------|
| `LinkedBlockingQueue` | 无界队列（默认 Integer.MAX_VALUE），可能 OOM |
| `ArrayBlockingQueue` | 有界队列，需指定容量 |
| `SynchronousQueue` | 不存储元素，直接传递，适合 CachedThreadPool |
| `PriorityBlockingQueue` | 带优先级的无界队列 |
| `DelayQueue` | 延迟队列，实现定时任务 |

### 5.3 四种拒绝策略

| 策略 | 说明 |
|------|------|
| `AbortPolicy`（默认） | 抛出 `RejectedExecutionException` |
| `CallerRunsPolicy` | 由提交任务的线程直接执行 |
| `DiscardPolicy` | 静默丢弃新任务 |
| `DiscardOldestPolicy` | 丢弃队列中最旧的任务 |

### 5.4 完整示例

```java
import java.util.concurrent.*;
import java.util.concurrent.atomic.AtomicInteger;

public class ThreadPoolDemo {

    public static void main(String[] args) throws InterruptedException {
        // 自定义线程工厂
        ThreadFactory namedFactory = new ThreadFactory() {
            private final AtomicInteger count = new AtomicInteger(1);
            @Override
            public Thread newThread(Runnable r) {
                Thread t = new Thread(r, "worker-" + count.getAndIncrement());
                t.setDaemon(false);
                return t;
            }
        };

        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            2,                           // corePoolSize
            4,                           // maximumPoolSize
            60L,                         // keepAliveTime
            TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(10), // 有界队列
            namedFactory,
            new ThreadPoolExecutor.CallerRunsPolicy()
        );

        for (int i = 0; i < 20; i++) {
            final int taskId = i;
            executor.submit(() -> {
                System.out.println(Thread.currentThread().getName() + " 执行任务 " + taskId);
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            });
        }

        executor.shutdown();
        executor.awaitTermination(10, TimeUnit.SECONDS);

        // 打印线程池统计信息
        System.out.println("完成任务数: " + executor.getCompletedTaskCount());
    }
}
```

### 5.5 线程池监控

```java
// 获取线程池运行状态
System.out.println("核心线程数: " + executor.getCorePoolSize());
System.out.println("当前线程数: " + executor.getPoolSize());
System.out.println("活跃线程数: " + executor.getActiveCount());
System.out.println("队列任务数: " + executor.getQueue().size());
System.out.println("已完成任务: " + executor.getCompletedTaskCount());
System.out.println("总任务数:   " + executor.getTaskCount());
```

## 六、常用线程池类型

> ⚠️ 阿里巴巴 Java 开发规范要求：**禁止使用 Executors 工具类创建线程池**，应使用 `ThreadPoolExecutor` 手动创建。

| 线程池 | 内部实现 | 适用场景 | 风险 |
|--------|----------|----------|------|
| `newFixedThreadPool(n)` | LinkedBlockingQueue（无界） | CPU 密集型任务 | 队列堆积 OOM |
| `newCachedThreadPool()` | SynchronousQueue | 大量短时任务 | 线程数无上限 OOM |
| `newSingleThreadExecutor()` | LinkedBlockingQueue（无界） | 顺序执行任务 | 队列堆积 OOM |
| `newScheduledThreadPool(n)` | DelayQueue | 定时/周期任务 | 无界队列风险 |
| `newWorkStealingPool()` | ForkJoinPool | 并行递归任务 | — |

## 七、线程池最佳实践

### 7.1 线程数配置公式

- **CPU 密集型**：线程数 = CPU 核数 + 1
- **IO 密集型**：线程数 = CPU 核数 × (1 + 等待时间/计算时间)，一般 2×CPU 核数
- **混合型**：根据 VisualVM / Arthas 实际分析

```java
int cpuCores = Runtime.getRuntime().availableProcessors();
// IO密集型示例
int threadCount = cpuCores * 2;
```

### 7.2 优雅关闭线程池

```java
public static void gracefulShutdown(ExecutorService executor) {
    executor.shutdown(); // 不接受新任务，等待队列任务完成
    try {
        if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
            executor.shutdownNow(); // 强制中断正在执行的任务
            if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
                System.err.println("线程池未正常关闭");
            }
        }
    } catch (InterruptedException e) {
        executor.shutdownNow();
        Thread.currentThread().interrupt();
    }
}
```

### 7.3 异常处理

```java
// 方式1：在任务内部捕获
executor.submit(() -> {
    try {
        doTask();
    } catch (Exception e) {
        log.error("任务执行异常", e);
    }
});

// 方式2：自定义 ThreadFactory，设置 UncaughtExceptionHandler
ThreadFactory factory = r -> {
    Thread t = new Thread(r);
    t.setUncaughtExceptionHandler((thread, e) ->
        log.error("线程 {} 发生未捕获异常", thread.getName(), e));
    return t;
};

// 方式3：继承 ThreadPoolExecutor，重写 afterExecute
public class MonitoredThreadPool extends ThreadPoolExecutor {
    @Override
    protected void afterExecute(Runnable r, Throwable t) {
        super.afterExecute(r, t);
        if (t == null && r instanceof Future<?>) {
            try {
                ((Future<?>) r).get();
            } catch (ExecutionException ee) {
                t = ee.getCause();
            } catch (InterruptedException ie) {
                Thread.currentThread().interrupt();
            }
        }
        if (t != null) {
            log.error("任务执行异常", t);
        }
    }
}
```

## 八、大厂常见面试题

### Q1：线程池的核心参数有哪些？各自的作用是什么？

**答：**

`ThreadPoolExecutor` 共有 7 个核心参数：

1. **`corePoolSize`（核心线程数）**：线程池维持的最小线程数，即使空闲也不会销毁（除非设置 `allowCoreThreadTimeOut(true)`）。
2. **`maximumPoolSize`（最大线程数）**：线程池能创建的最大线程数，超过此值执行拒绝策略。
3. **`keepAliveTime`（空闲存活时间）**：非核心线程空闲超过此时间后被销毁。
4. **`unit`（时间单位）**：配合 `keepAliveTime` 使用。
5. **`workQueue`（工作队列）**：存放等待执行任务的阻塞队列。
6. **`threadFactory`（线程工厂）**：用于创建线程，可自定义线程名、优先级等。
7. **`handler`（拒绝策略）**：队列满且线程数达上限时的处理策略。

---

### Q2：线程池的任务提交流程是什么？

**答：**

1. 若当前线程数 < `corePoolSize`，直接创建核心线程执行任务。
2. 若当前线程数 >= `corePoolSize`，尝试将任务放入 `workQueue`。
3. 若队列已满，且当前线程数 < `maximumPoolSize`，创建非核心线程执行任务。
4. 若队列已满，且当前线程数 >= `maximumPoolSize`，执行拒绝策略。

> 注意：任务并**不是**优先放入队列再创建线程，而是先尝试核心线程，核心线程满才入队。

---

### Q3：为什么阿里规范禁止使用 Executors 创建线程池？

**答：**

- `newFixedThreadPool` 和 `newSingleThreadExecutor` 使用的是 `LinkedBlockingQueue`，队列长度为 `Integer.MAX_VALUE`，大量任务堆积时可能导致 **OOM**。
- `newCachedThreadPool` 最大线程数为 `Integer.MAX_VALUE`，高并发下可能创建大量线程，耗尽系统资源导致 **OOM**。
- `newScheduledThreadPool` 同样使用无界队列，存在堆积风险。

应使用 `ThreadPoolExecutor` 手动指定有界队列和合理的参数，并自定义拒绝策略。

---

### Q4：`shutdown()` 和 `shutdownNow()` 的区别？

**答：**

| 方法 | 说明 |
|------|------|
| `shutdown()` | 平滑关闭：不再接受新任务，等待已提交任务（含队列中）执行完成 |
| `shutdownNow()` | 立即关闭：中断正在执行的线程，返回队列中尚未执行的任务列表 |

生产中推荐先 `shutdown()`，超时后再 `shutdownNow()`。

---

### Q5：线程池如何实现动态调整线程数？

**答：**

`ThreadPoolExecutor` 提供了以下方法支持运行时调整：

```java
// 动态修改核心线程数
executor.setCorePoolSize(newCoreSize);
// 动态修改最大线程数
executor.setMaximumPoolSize(newMaxSize);
// 允许核心线程超时
executor.allowCoreThreadTimeOut(true);
```

美团开源的 **动态线程池**（DynamicTp）框架即基于此实现，通过配置中心（如 Nacos）实时下发配置变更，结合 Actuator 监控线程池健康状态，是生产级线程池治理的最佳实践。
