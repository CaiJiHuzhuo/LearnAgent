## 一、CountDownLatch

### 1.1 核心用法

`CountDownLatch` 让一个或多个线程等待其他线程完成操作后再继续执行。内部维护一个计数器，`countDown()` 减 1，`await()` 等待计数器归 0。

**不可重用**：一旦计数器为 0，不能重置（如需复用请用 `CyclicBarrier`）。

### 1.2 场景一：主线程等待多个子任务

```java
import java.util.concurrent.*;

public class ParallelTaskDemo {
    public static void main(String[] args) throws InterruptedException {
        int taskCount = 5;
        CountDownLatch latch = new CountDownLatch(taskCount);
        ExecutorService pool = Executors.newFixedThreadPool(taskCount);

        long start = System.currentTimeMillis();
        for (int i = 0; i < taskCount; i++) {
            final int id = i;
            pool.submit(() -> {
                try {
                    System.out.println("任务 " + id + " 开始");
                    Thread.sleep((long)(Math.random() * 1000));
                    System.out.println("任务 " + id + " 完成");
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                } finally {
                    latch.countDown(); // 无论成功失败都要 countDown
                }
            });
        }

        latch.await(); // 等待所有任务完成
        System.out.println("所有任务完成，耗时: " + (System.currentTimeMillis() - start) + "ms");
        pool.shutdown();
    }
}
```

### 1.3 场景二：多线程同时开始（发令枪）

```java
public class StartingGunDemo {
    public static void main(String[] args) throws InterruptedException {
        int runnerCount = 5;
        CountDownLatch readyLatch = new CountDownLatch(runnerCount); // 等待运动员就绪
        CountDownLatch startLatch = new CountDownLatch(1);           // 发令枪

        for (int i = 0; i < runnerCount; i++) {
            final int id = i;
            new Thread(() -> {
                try {
                    System.out.println("运动员 " + id + " 就绪");
                    readyLatch.countDown();
                    startLatch.await(); // 等待发令
                    System.out.println("运动员 " + id + " 起跑！");
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }).start();
        }

        readyLatch.await(); // 等待所有运动员就绪
        System.out.println("发令！");
        startLatch.countDown(); // 开跑
    }
}
```

## 二、CyclicBarrier

### 2.1 核心特性

`CyclicBarrier` 让一组线程互相等待，直到所有线程都到达屏障点，然后一起继续执行。与 `CountDownLatch` 最大的区别是**可循环复用**。

```java
import java.util.concurrent.*;

public class CyclicBarrierDemo {
    public static void main(String[] args) {
        int parties = 3;
        // 所有线程到达后，先执行 barrierAction，再放行所有线程
        CyclicBarrier barrier = new CyclicBarrier(parties,
            () -> System.out.println("【所有线程到达屏障，开始下一阶段】"));

        for (int i = 0; i < parties; i++) {
            final int id = i;
            new Thread(() -> {
                try {
                    // 第一阶段
                    System.out.println("线程 " + id + " 完成第一阶段");
                    barrier.await(); // 等待其他线程

                    // 第二阶段
                    System.out.println("线程 " + id + " 完成第二阶段");
                    barrier.await(); // 屏障可复用！

                    System.out.println("线程 " + id + " 全部完成");
                } catch (InterruptedException | BrokenBarrierException e) {
                    Thread.currentThread().interrupt();
                }
            }).start();
        }
    }
}
```

### 2.2 CyclicBarrier 实现分布式计算

```java
// 模拟将大数据集分段处理后汇总
public class MatrixCompute {
    private static final int THREAD_COUNT = 4;
    private static final int[][] matrix = new int[THREAD_COUNT][1000];
    private static final int[] partialResults = new int[THREAD_COUNT];

    public static void main(String[] args) throws InterruptedException, BrokenBarrierException {
        CyclicBarrier barrier = new CyclicBarrier(THREAD_COUNT, () -> {
            // 汇总所有分段结果
            int total = 0;
            for (int r : partialResults) total += r;
            System.out.println("最终结果: " + total);
        });

        for (int i = 0; i < THREAD_COUNT; i++) {
            final int segment = i;
            new Thread(() -> {
                // 计算本段
                int sum = 0;
                for (int v : matrix[segment]) sum += v;
                partialResults[segment] = sum;
                System.out.println("段 " + segment + " 计算完成: " + sum);
                try {
                    barrier.await();
                } catch (InterruptedException | BrokenBarrierException e) {
                    Thread.currentThread().interrupt();
                }
            }).start();
        }
    }
}
```

## 三、Semaphore

### 3.1 控制并发量

`Semaphore` 可以控制同时访问特定资源的线程数量，类似操作系统的信号量机制。

```java
import java.util.concurrent.*;

// 数据库连接池模拟（最多3个并发连接）
public class ConnectionPoolDemo {
    private static final Semaphore semaphore = new Semaphore(3);
    private static final String[] DB_CONNECTIONS = {"conn-1", "conn-2", "conn-3"};
    private static final Queue<String> connPool = new LinkedBlockingQueue<>(Arrays.asList(DB_CONNECTIONS));

    public static String acquire() throws InterruptedException {
        semaphore.acquire();
        return connPool.poll();
    }

    public static void release(String conn) {
        connPool.offer(conn);
        semaphore.release();
    }

    public static void main(String[] args) {
        ExecutorService pool = Executors.newFixedThreadPool(10);
        for (int i = 0; i < 10; i++) {
            final int id = i;
            pool.submit(() -> {
                String conn = null;
                try {
                    conn = acquire();
                    System.out.println("线程 " + id + " 使用连接: " + conn);
                    Thread.sleep(500); // 模拟查询
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                } finally {
                    if (conn != null) release(conn);
                }
            });
        }
        pool.shutdown();
    }
}
```

### 3.2 tryAcquire 非阻塞使用

```java
// 带超时的尝试获取
if (semaphore.tryAcquire(100, TimeUnit.MILLISECONDS)) {
    try {
        // 使用资源
    } finally {
        semaphore.release();
    }
} else {
    System.out.println("获取许可超时，执行降级逻辑");
}
```

## 四、Exchanger

`Exchanger` 用于两个线程之间交换数据，当两个线程都到达交换点时，交换各自持有的数据。

```java
import java.util.concurrent.Exchanger;

public class ExchangerDemo {
    public static void main(String[] args) {
        Exchanger<String> exchanger = new Exchanger<>();

        // 生产者
        new Thread(() -> {
            try {
                String data = "生产者的数据";
                System.out.println("生产者发送: " + data);
                String received = exchanger.exchange(data); // 阻塞，等待另一方
                System.out.println("生产者收到: " + received);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }).start();

        // 消费者
        new Thread(() -> {
            try {
                Thread.sleep(1000); // 模拟延迟
                String data = "消费者的数据";
                System.out.println("消费者发送: " + data);
                String received = exchanger.exchange(data);
                System.out.println("消费者收到: " + received);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }).start();
    }
}
```

**典型应用：** 遗传算法中两个种群交换数据；双缓冲（生产线程填充一个缓冲区，消费线程处理另一个，通过 Exchanger 交换）。

## 五、Phaser

`Phaser`（阶段同步器）是 JDK 7 新增的，比 `CountDownLatch` 和 `CyclicBarrier` 更灵活，支持动态注册参与方。

```java
import java.util.concurrent.Phaser;

public class PhaserDemo {
    public static void main(String[] args) {
        // 注册1个参与方（主线程）
        Phaser phaser = new Phaser(1) {
            // 每个阶段完成时回调
            @Override
            protected boolean onAdvance(int phase, int registeredParties) {
                System.out.println("=== 第 " + phase + " 阶段完成，参与方: " + registeredParties + " ===");
                return registeredParties == 0; // 返回true表示终止
            }
        };

        // 动态注册和注销
        for (int i = 0; i < 3; i++) {
            phaser.register(); // 注册新参与方
            final int id = i;
            new Thread(() -> {
                for (int phase = 0; phase < 3; phase++) {
                    System.out.println("线程 " + id + " 完成第 " + phase + " 阶段");
                    phaser.arriveAndAwaitAdvance(); // 到达并等待
                }
                phaser.arriveAndDeregister(); // 完成后注销
            }).start();
        }

        // 主线程参与每个阶段
        for (int phase = 0; phase < 3; phase++) {
            phaser.arriveAndAwaitAdvance();
        }
        phaser.arriveAndDeregister();
    }
}
```

## 六、BlockingQueue

阻塞队列是线程安全的队列，提供了阻塞的入队和出队操作。

### 6.1 核心方法对比

| 操作 | 抛异常 | 返回特殊值 | 阻塞 | 超时阻塞 |
|------|--------|----------|------|---------|
| 入队 | `add(e)` | `offer(e)` | `put(e)` | `offer(e, t, unit)` |
| 出队 | `remove()` | `poll()` | `take()` | `poll(t, unit)` |
| 查看 | `element()` | `peek()` | — | — |

### 6.2 常用实现类

```java
import java.util.concurrent.*;

// 1. ArrayBlockingQueue：有界，公平/非公平
BlockingQueue<String> arrayQueue = new ArrayBlockingQueue<>(100, true);

// 2. LinkedBlockingQueue：可选有界（默认 Integer.MAX_VALUE）
BlockingQueue<String> linkedQueue = new LinkedBlockingQueue<>(1000);

// 3. PriorityBlockingQueue：无界优先级队列
BlockingQueue<Integer> priorityQueue = new PriorityBlockingQueue<>();

// 4. DelayQueue：延迟队列，元素须实现 Delayed 接口
BlockingQueue<DelayedTask> delayQueue = new DelayQueue<>();

// 5. SynchronousQueue：没有容量，每次入队必须等待出队
BlockingQueue<String> syncQueue = new SynchronousQueue<>();

// 6. LinkedTransferQueue：更高性能的无界传输队列
BlockingQueue<String> transferQueue = new LinkedTransferQueue<>();
```

### 6.3 生产者-消费者模式

```java
import java.util.concurrent.*;

public class ProducerConsumer {
    private static final BlockingQueue<Integer> queue = new LinkedBlockingQueue<>(50);

    // 生产者
    static class Producer implements Runnable {
        @Override
        public void run() {
            try {
                for (int i = 0; i < 100; i++) {
                    queue.put(i);        // 队列满则阻塞
                    System.out.println("生产: " + i + ", 队列大小: " + queue.size());
                    Thread.sleep(10);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }

    // 消费者
    static class Consumer implements Runnable {
        @Override
        public void run() {
            try {
                while (true) {
                    Integer item = queue.poll(2, TimeUnit.SECONDS); // 等待2秒
                    if (item == null) break; // 超时则退出
                    System.out.println("消费: " + item);
                    Thread.sleep(50);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }

    public static void main(String[] args) throws InterruptedException {
        ExecutorService pool = Executors.newFixedThreadPool(3);
        pool.submit(new Producer());
        pool.submit(new Consumer());
        pool.submit(new Consumer());
        pool.shutdown();
        pool.awaitTermination(30, TimeUnit.SECONDS);
    }
}
```

## 七、ConcurrentHashMap

### 7.1 JDK 7 vs JDK 8 实现对比

| 版本 | 底层实现 | 加锁粒度 |
|------|----------|----------|
| JDK 7 | Segment 数组 + HashEntry 链表 | Segment 级别（分段锁，16 个 Segment） |
| JDK 8 | Node 数组 + 链表/红黑树 | 桶（数组槽）级别（CAS + synchronized） |

### 7.2 JDK 8 关键操作

```java
import java.util.concurrent.ConcurrentHashMap;

ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

// 普通操作（线程安全）
map.put("key", 1);
map.get("key");

// 原子复合操作（避免 check-then-act 竞态）
map.putIfAbsent("key", 1);               // 不存在才放入
map.computeIfAbsent("key", k -> 1);      // 不存在才计算并放入
map.computeIfPresent("key", (k, v) -> v + 1); // 存在才更新
map.compute("key", (k, v) -> v == null ? 1 : v + 1); // 统计词频
map.merge("key", 1, Integer::sum);       // 合并（等同于上面的 compute 统计词频）

// 注意：size() 只是近似值（高并发下不精确），精确用 mappingCount()
long count = map.mappingCount();
```

### 7.3 JDK 8 put 流程

```
put(key, value)
    │
    ▼
计算 hash，定位数组下标 i
    │
    ├─ 数组[i] == null
    │      └─► CAS 插入新节点（无锁）
    │
    ├─ 数组[i].hash == MOVED（正在扩容）
    │      └─► helpTransfer 协助扩容
    │
    └─ 数组[i] 不为空
           └─► synchronized(数组[i]) 加桶头锁
                   ├─ 链表：遍历更新或尾插
                   └─ 红黑树：树插入
                   
    插入后检查链表长度，>= 8 时转红黑树（treeifyBin）
```

## 八、大厂常见面试题

### Q1：CountDownLatch 和 CyclicBarrier 有什么区别？

**答：**

| 维度 | CountDownLatch | CyclicBarrier |
|------|---------------|---------------|
| 等待方向 | 一个（或多个）线程等待其他 N 个线程 | N 个线程互相等待（到达屏障） |
| 可重置 | 不可重置，一次性 | 可调用 `reset()` 循环复用 |
| 计数主体 | `countDown()` 可由任意线程调用 | 必须所有参与方都调用 `await()` |
| 屏障行动 | 无 | 可设置 `barrierAction`，所有线程到达后执行 |
| 底层实现 | AQS 共享锁 | ReentrantLock + Condition |

典型场景：`CountDownLatch` 适合等待多个子任务完成（类似 `join`）；`CyclicBarrier` 适合分阶段并行计算（如 MapReduce 的多轮迭代）。

---

### Q2：ConcurrentHashMap 在 JDK 8 中如何实现线程安全？

**答：**

JDK 8 的 `ConcurrentHashMap` 放弃了 JDK 7 的 Segment 分段锁，改用更细粒度的方式：

1. **CAS 操作**：初始化数组、插入空桶位时，使用 CAS 保证原子性，无需加锁。
2. **synchronized 桶头锁**：当目标桶不为空时，对桶头节点（`Node`）加 `synchronized` 锁，只锁住一个桶，粒度比 Segment 更细。
3. **volatile 变量**：`Node` 的 `val` 和 `next` 字段都是 volatile，保证可见性。
4. **扩容时的协助迁移**：多线程可同时参与扩容，提升扩容效率。

---

### Q3：BlockingQueue 的 put 和 offer 有什么区别？

**答：**

| 方法 | 队列满时行为 |
|------|------------|
| `put(e)` | **一直阻塞**，直到有空间（可被中断） |
| `offer(e)` | 立即返回 false，不阻塞 |
| `offer(e, timeout, unit)` | 等待指定时间，超时返回 false |
| `add(e)` | 抛出 `IllegalStateException` |

生产者-消费者模式中推荐用 `put`（保证不丢数据）或带超时的 `offer`（有降级逻辑时）。

### Q4：Semaphore 能实现互斥锁吗？

**答：**

可以。将 `Semaphore` 初始化为 1（二值信号量），就相当于一个互斥锁：

```java
Semaphore mutex = new Semaphore(1);
mutex.acquire(); // 加锁
try {
    // 临界区
} finally {
    mutex.release(); // 解锁
}
```

但与 `ReentrantLock` 不同，Semaphore **不可重入**（同一线程调用两次 acquire 会死锁），且可以由**不同线程**来 release（Semaphore 不绑定所有者线程），这使得它也可以实现跨线程的资源释放（如线程B 释放线程A 占用的许可）。

### Q5：如何实现一个高性能的有界阻塞队列？

**答：**

可以基于 `ReentrantLock` + 两个 `Condition` 手动实现，类似 JDK 的 `ArrayBlockingQueue`：

```java
public class BoundedBlockingQueue<T> {
    private final Object[] items;
    private int head, tail, count;
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notEmpty = lock.newCondition();
    private final Condition notFull  = lock.newCondition();

    public BoundedBlockingQueue(int capacity) {
        items = new Object[capacity];
    }

    public void put(T e) throws InterruptedException {
        lock.lockInterruptibly();
        try {
            while (count == items.length) notFull.await();
            items[tail] = e;
            if (++tail == items.length) tail = 0;
            count++;
            notEmpty.signal();
        } finally {
            lock.unlock();
        }
    }

    @SuppressWarnings("unchecked")
    public T take() throws InterruptedException {
        lock.lockInterruptibly();
        try {
            while (count == 0) notEmpty.await();
            T e = (T) items[head];
            items[head] = null;
            if (++head == items.length) head = 0;
            count--;
            notFull.signal();
            return e;
        } finally {
            lock.unlock();
        }
    }
}
```

高性能改进方向：使用 `LinkedTransferQueue`（无锁设计）、Disruptor 环形缓冲区（消除伪共享、无锁）等。
